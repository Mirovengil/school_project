<p>Итак, вам понадобилось удалить один элемент из массива. Этот алгоритм
можно реализовать несколькими способами, которые мы разберём ниже.</p>
<h2>Без сохранения порядка</h2>
<p>Характеристика реализации:
<ul>
    <li>Проста в написании;</li>
    <li>Быстра;</li>
    <li>Нарушает порядок элементов в массиве;</li>
</ul>
В принципе, если массив неупорядочен, то сохранение, собственно, порядка
не имеет смысла. Поэтому реализация не рекомендуется к использованию на
отсортированных массивах, потому что придётся выполнять новую сортировку,
что может быть весьма затратно по времени.
</p>
<p>Реализация заключается в следующем: надо взять элемент, обменять его 
значениями с последним элементом массива и уменьшить длину последнего на единицу.</p>
<p>Если вы не знаете, как обменять значениями две переменные, рекомендую посмотреть
соответствующую статью:</p>
<a href="./swap.html" class = "btn">Обмен двух переменных значениями</a>
<p>На всякий случай -- иллюстрации:</p>
<img src="./img/arrays/015.png">
<img src="./img/arrays/016.png">
<img src="./img/arrays/017.png">
<img src="./img/arrays/018.png">
<p>Как вы видите, алгоритм совершенно несложен. Давайте теперь его запишем
на языках программирования, используя статический (динамические в средней школе
не проходят) массив <b>arr</b> и переменную его длины <b>n</b> -- удалим элемент
с индексом <b>to_delete</b>:</p>

<~c~>
//Меняем значениями удаляемый и последний
//элементы массива:
int temp = arr[to_delete];
arr[to_delete] = arr[n - 1];
arr[n - 1] = temp;
//Удаляем последний элемент:
n -= 1;
<~/c~>

<~pa~>
//Меняем значениями удаляемый и последний
//элементы массива:
temp := arr[to_delete];
arr[to_delete] := arr[n - 1];
arr[n - 1] := temp;
//Удаляем последний элемент:
n := n - 1;
<~/pa~>

<h2>С сохранением порядка</h2>

<p>Характеристика реализации:
<ul>
    <li>Немного сложнее для понимания;</li>
    <li>Дольше работает (в олимпиаде может быть важным);</li>
    <li>Сохраняет порядок элементов в массиве;</li>
</ul></p>

<p>Эта реализация заключается в том, что часть массива после удаляемого элемента
последовательно "сползает" на один элемент, перекрывая удаляемый элемент. Тут без
иллюстраций обойтись не получится:</p>

<p>Как вы можете видеть, остальные элементы сохранили свой порядок. Теперь реализация.
Мы используем статический (динамические в средней школе
не проходят) массив <b>arr</b> и переменную его длины <b>n</b> -- удалим элемент
с индексом <b>to_delete</b>: </p>
<~c~>
//"Стягиваем" все элементы массива на индекс назад
//Обратите внимание: так как нам необходимо обращаться к следующему за
//i-ным элементу, то i + 1 должно быть допустимым.
for (int i = to_delete; i + 1 < n; ++i)
    arr[i] = arr[i + 1];
//Уменьшаем размер массива на единицу.
n -= 1;
<~/c~>

<~pa~>
{"Стягиваем" все элементы массива на индекс назад}
{Обратите внимание: так как нам необходимо обращаться к следующему за}
{i-ным элементу, то i + 1 должно быть допустимым.}
{i <= n - 2 равносильно i + 1 <= n - 1, то есть, это гарантия, что}
{не будет совершено обращение к элементу, находящемуся вне массива.}
for i := to_delete to n - 2 do
    arr[i] := arr[i + 1];
n := n - 1;
{Уменьшаем размер массива на единицу.}
<~/pa~>
<h2>Встроенными средствами языка</h2>
<p>Внимательный читатель, вероятно, уже задался вопросом, где же примеры
на языке Python? Дело в том, что в Python'е внутри языка реализован
очень большой набор функций, среди которых есть и удаление элемента из массива.</p>
<p>Мы используем массив <b>arr</b> -- удалим элемент с индексом <b>to_delete</b>: </p>
<~py~>arr.pop(to_delete)<~/py~>
<p>Вот, в принципе, и всё. Удаление элемента из массива -- задача несложная, но уметь её 
выполнять должен каждый, кто изучает программирование. Спасибо, что читаете нас!</p>
